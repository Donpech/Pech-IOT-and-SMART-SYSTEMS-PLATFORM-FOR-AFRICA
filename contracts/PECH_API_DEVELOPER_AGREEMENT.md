## 3. PERMITTED USE

3.1. The Developer may use the PECH APIs to:

   (a) Build applications, integrations, and plugins that interact with the PECH Platform;

   (b) Access Sandbox environments for testing and development;

   (c) Process transactions and data through the PECH Platform as documented;

   (d) Display PECH-sourced data within the Developer Application in compliance with PECH's branding guidelines.

3.2. The Developer shall comply with all applicable API rate limits, usage quotas, and version requirements as specified in the Documentation.

---

## 4. API KEYS AND CREDENTIALS

4.1. PECH shall issue API Keys to the Developer upon approval of this Agreement.

4.2. The Developer shall:

   (a) Keep all API Keys and credentials strictly confidential;

   (b) Not share, publish, or expose API Keys in client-side code, public repositories, or any publicly accessible location;

   (c) Implement industry-standard security measures to protect API Keys;

   (d) Notify PECH immediately if any API Key is compromised or suspected to be compromised;

   (e) Rotate API Keys as recommended by PECH.

4.3. PECH may revoke API Keys immediately if they are compromised, misused, or if the Developer breaches this Agreement.

---

## 5. DEVELOPER OBLIGATIONS

5.1. The Developer shall:

   (a) Comply with all PECH API Documentation, guidelines, and best practices;

   (b) Maintain compatibility with the current supported version of the PECH APIs;

   (c) Implement appropriate error handling and rate limit compliance;

   (d) Maintain reasonable uptime and performance standards for the Developer Application;

   (e) Provide accurate and up-to-date contact information to PECH;

   (f) Respond promptly to PECH communications regarding security issues or API changes;

   (g) Display appropriate PECH attribution as required by the Documentation.

---

6.1. **PECH IP:** PECH retains all right, title, and interest in and to the Platform, APIs, SDKs, Documentation, trademarks, and all related Intellectual Property. Nothing in this Agreement transfers any PECH IP to the Developer.

6.2. **Developer IP:** The Developer retains ownership of the Developer Application code that is independently created, excluding any PECH IP incorporated therein.

6.3. **Contributions:** Any feedback, suggestions, bug reports, or contributions submitted by the Developer to PECH regarding the Platform or APIs shall become the exclusive property of PECH, and the Developer hereby assigns all rights therein to PECH.

6.4. **Showcase License:** The Developer grants PECH a non-exclusive, royalty-free, worldwide license to list, showcase, and promote the Developer Application on PECH's website, documentation, and marketing materials.

6.5. **No Reverse Engineering:** The Developer shall not reverse engineer, decompile, disassemble, or attempt to derive the source code of the PECH APIs or Platform.

## 7. DATA PROTECTION

7.1. The Developer shall comply with the Nigeria Data Protection Act 2023 (NDPA) and all applicable data protection laws when processing User Data via PECH APIs.

7.2. The Developer shall:

   (a) Implement appropriate technical and organizational measures to protect User Data;

   (b) Process User Data only for the purposes disclosed to and consented by the end user;

   (c) Not sell, share, or transfer User Data to third parties without appropriate consent and legal basis;

   (d) Maintain a privacy policy that accurately describes data collection and processing practices;

   (e) Promptly notify PECH of any data breach affecting User Data within twenty-four (24) hours of discovery.

7.3. A Data Processing Agreement is attached as **Schedule A** and forms part of this Agreement.

## 9. CONFIDENTIALITY

9.1. The Developer shall keep confidential all non-public information received from PECH, including:

   (a) API documentation not publicly available;

   (b) Sandbox data and test environments;

   (c) Pre-release API features and roadmap information;

   (d) Technical specifications and architecture details;

   (e) Business terms, pricing, and revenue data.

9.2. Confidentiality obligations shall survive termination of this Agreement for a period of five (5) years, and indefinitely for trade secrets.

## 10. ACCEPTABLE USE POLICY

10.1. The Developer shall NOT:

   (a) Use the PECH APIs to build a competing platform or service;

   (b) Scrape, crawl, or mine data from the PECH Platform beyond API access;

   (c) Attempt to circumvent API rate limits, authentication, or security measures;

   (d) Use the APIs for any illegal, fraudulent, or harmful purpose;

   (e) Distribute malware, viruses, or malicious code through the Developer Application;

   (f) Misrepresent the Developer Application as an official PECH product;

   (g) Use PECH trademarks or branding in a way that implies endorsement without written approval;

   (h) Store or cache User Data beyond what is necessary for the Developer Application's functionality;

   (i) Facilitate or enable unauthorized access to the PECH Platform.

11.1. PECH shall use reasonable efforts to maintain API availability but provides the APIs **"as-is"** for the free tier.

11.3. PECH shall provide reasonable notice of scheduled maintenance and API changes.

11.4. PECH shall not be liable for any downtime, data loss, or service interruptions beyond the scope of any applicable SLA.

## 12. TERMINATION

12.1. **Developer Termination:** The Developer may terminate this Agreement at any time by providing thirty (30) days' written notice to PECH and ceasing use of the APIs.

12.2. **PECH Termination:** PECH may terminate this Agreement or revoke API access:

   (a) Immediately, for breach of the Acceptable Use Policy or security incidents;

   (b) With thirty (30) days' notice, for any other reason.

12.3. **Effect of Termination:**

   (a) All API access shall be revoked;

   (b) The Developer shall cease using all PECH APIs, SDKs, and Documentation;

   (c) The Developer shall delete all API Keys, cached data, and PECH Confidential Information;

   (d) Revenue sharing obligations for earned but unpaid amounts shall survive;

   (e) Clauses 6, 7, 9, 10, 13, 14, 15, and 16 shall survive termination.

12.4. **Data Portability:** The Developer may export their own data within thirty (30) days of termination.

   (d) Any data breach or security incident caused by the Developer;
